术语表是用于 OmegaT 的文件，它由用户手工创建和更新。

如果某个 OmegaT 项目包含一个或多个术语表文件，当前片段中出现的任何术语将自动显示在术语表查看器中。

您可以在项目属性对话框中设定其位置和名称。扩展名必须为.txt or .utf8 （如果不是的话，加上扩展名）. 文件必须保存在 /glossary文件夹中，但它可以保存在/glossary的子文件夹中，如, glossary/sub/glossary.txt). 在设置时该文件可以不存在，在添加词条时会自动创建该文件（如果需要）。如果该文件已存在，不会对其格式或字符集进行检查：新条目总是以tab分隔并使用UTF-8编码。由于不会操作已有的内容，这样对已有文件的破坏则受到了限制。

1. 用法

要使用现有的术语表，只需在创建项目后将它放到 /glossary 目录下。在打开项目时，OmegaT 自动检测该目录下的术语表文件。从术语表中找到当前片段中的术语后，OmegaT 将会在术语表窗格将它们显示出来。

2. 文件格式

术语表文件是简单的纯文本文件，包含了以制表符分隔的三列列表，其中第一和第二列分别对应源词汇和目标术语。第三列用来保存附加信息。您可以添加目标列为空的条目，即只包含源术语和注释。

术语表的编码是自动决定的。当然，编码必须和术语语言兼容，因此建议使用Unicode码。

还支持 CSV 格式。该格式和 TAB 分隔的相同：源术语，目标术语。注释字段使用逗号 ',' 分隔开来。字符串需要括在引号 " 里面，这样可以在字符串中使用逗号：

"This is a source term, which contains a comma","c'est un terme, qui contient une virgule"

除了纯文本格式， 也支持TBX格式的只读术语。.tbx 文件的地址必须在 /glossary文件夹中，但也能存在其子文件夹下，如 glossary/sub/MyGlossary.tbx。).

TBX（Term Base eXchange）是用于交换结构化术语数据的基于 XML 的开放标准，已经被LISA和ISO批准为国际标准。如果你已经有术语处理系统，那它很可能提供通过 TBX格式到处术语数据。. 可以下载Microsoft Terminology Collection 它有将近100种语言，可以作为基础的IT词汇表。

注意：MultiTerm导出的.tbx格式文件似乎不太可靠（2013年11月），建议使用MultiTerm的.tab文件。

3. 如何创建术语表

你可以在项目设置时输入术语表文件的名字来创建可写入术语表（见本章开端）。. 在术语表窗格点击右键或按下Ctrl+Shift+G来添加新条目。在打开的对话框中可以输入源术语、目标术语及注释。

术语表文件的内容会存放到内存中，在打开或重新载入项目时加载它们。因此更新术语表变得非常简单：按下Ctrl+Shift+G并输入新术语、其翻译及注释（确保在字段间使用了TAB）并保存文件。因此术语表窗格的内容将会被更新。

可写术语表的位置可在项目>属性...对话框中设定。可识别出的扩展名为TXT和UTF8

注：还有其他创建含TAB分隔条目的简单文件的方法。在Windows平台上使用 Notepad++，在Linux平台上使用 GEdit， 或者用电子表格完成都是可以的：任何能处理 UTF-8 (或者UTF-16 LE) ，以及能能显示空格（你不会漏掉必要的 TAB 字符）的应用都可以使用。

4. 优先术语

优先术语表 (默认的glossary/glossary.txt) 结果会在术语窗口和翻译建议中中优先出现。

优先术语表和非优先术语表中的术语可能混合，优先术语表中的词汇以粗体展现。

5. 使用Trados MultiTerm

在 Trados MultiTerm 导出的数据可以不加修改地用作 OmegaT 术语表，只需以 .tab为扩展名，并让对应的源词汇和目标术语分别为头两个字段。如果使用 "Tab-delimited export" 系统选项导出，您必须删除头 5 列（包括编号, 创建日期等等）。

6. 常见术语表问题

问题：未显示术语表术语——可能的解决方法：

  • "glossary" 目录中没有找到术语表文件。

  • 术语表文件为空。

  • 词条的各项间未使用 TAB 字符分隔。

  • 术语表文件没有正确的拓展名 (.tab, .utf8 or .txt)。

  • 术语表词条和文档中的原文未完全匹配－－例如该词条是复数形式。

  • 术语表文件没有采用正确的编码。

  • 当前片段中没有任何术语与术语表中的术语相匹配。

  • 上述一个或多个问题已经被修正，但项目还没有被重新载入。

问题：在术语表窗格中某些字符显示不正确。
