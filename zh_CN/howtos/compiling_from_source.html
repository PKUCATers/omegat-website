Howto - 从源码编译OmegaT

这篇如何描述如何从源代码编译OmegaT。

OmegaT是为许多平台提供的现成编译版本，包括跨平台版本。用户因此不需要自己编译OmegaT。编译OmegaT对希望对自己的代码进行修改的开发人员很重要。

1. 下载并安装Java开发工具包（JDK )

Java开发工具包是必需的。对于OmegaT 4.0及更高版本，不需要额外的软件。

早期版本的OmegaT需要 Apache Ant 。

您可能必须将您的 JAVA_HOME 环境变量设置为指向JDK的位置。

2. 编译代码

打开命令行窗口（控制台，DOS窗口）。例如，导航到OmegaT源代码的根目录（包含文件 build.gradle 的文件夹）。

cd C：\ My Documents \ MyOmegat \ OmegaT_ <version> _Beta_Source

OmegaT 4.0或更高版本

您可以直接从命令行运行OmegaT

gradlew运行

如果这是你感兴趣的所有内容，你可以跳到步骤3a。

而不是建立OmegaT程序和所有相关的文件。

gradlew installDist

结果将在 build \ install \ OmegaT </ c0>中显示。

旧版本的OmegaT

从根文件夹中，通过输入ant可执行文件来执行Apache Ant。可能需要输入完整路径，例如：

C：\ Program Files \ apache-ant-1.7.0-bin \ apache-ant-1.7.0 \ bin \ ant

如果代码编译成功，您将看到 BUILD SUCCESSFUL </ c0>消息。如果没有，则会显示调试消息，它将（希望）提供有关问题出现的一些信息。

如果你的代码编译成功，编译后的OmegaT被放入：

OmegaT_<version>_Beta_Source\dist

仍然在您的命令行窗口中，切换到此文件夹，例如有：

cd C:\My Documents\MyOmegat\OmegaT_<version>_Beta_Source\dist

并推出OmegaT：

 java -jar OmegaT.jar

3a. 第一个示例修改：更改段高亮的颜色

在文件夹 src / org / omegat / gui / editor </ c0>中，在文本编辑器中打开文件 SegmentBuilder.java </ c1>。

查找到以下行：

return Styles.createAttributeSet(fg, Styles.COLOR_GREEN, true, null);

and

b = Styles.COLOR_GREEN;

并用 COLOR_BLUE </ c1>替换 COLOR_GREEN </ c0>。

在文件夹 src / org / omegat / util / gui </ c0>中，在文本编辑器中打开文件 Styles.java </ c1>。

找到该行

public static final Color COLOR_GREEN = new Color（192，255，192）;

用 225,225,255 </ c3>替换 COLOR_GREEN </ c0>与 COLOR_BLUE </ c1>和 192,255,192 </ c2>。

3b. 第二个示例修改：将显示的模糊匹配数量从五个增加到十个

在文件夹 / src / org / omegat / util </ c0>中，在文本编辑器中打开文件 OConsts.java </ c1>。

查找到以下行：

    /** Number of fuzzy matches to display */
    public static final int MAX_NEAR_STRINGS = 5;

并用 = 10 </ c1>替换 = 5 </ c0>。

对于这两个示例，如上所述进行编译和测试。
